# Proxy Monitor - 极速健壮性增强与质量保障计划

## 愿景 (Vision)
为 Windows 用户提供一个零缺陷、高度可靠且具备极佳反馈机制的代理流量监控工具。确保程序在各种复杂环境（如网络不稳定、环境依赖缺失、配置异常）下都能稳定运行或提供清晰的排错指引。

## 核心价值 (Core Value)
- **极致稳定性：** 通过代码重构与自动化测试排除潜在逻辑漏洞。
- **透明的反馈：** 确保用户在遇到任何环境或网络问题时，能立即得到直观、易懂的反馈。
- **易维护性：** 将高度耦合的脚本拆分为模块，降低未来功能扩展的难度。

## 成功指标 (Success Criteria)
- [ ] **代码模块化：** 完成从单文件脚本到模块化结构的重组（配置类、网络类、UI类分离）。
- [ ] **高测试覆盖率：** 核心逻辑（配置读写、流量数据解析、预警判定）具备 90% 以上的单元测试覆盖率。
- [ ] **异常反馈闭环：** 验证所有关键失败路径（网络超时、订阅链接失效、依赖导入失败）均有相应的 UI 弹窗或日志提示。
- [ ] **环境健壮性：** 实现更完善的启动自检，能够智能处理及报告环境依赖问题。

## 需求 (Requirements)

### 已定义 (Validated)
- ✓ **基础监控：** 定期从订阅链接抓取流量数据。
- ✓ **系统集成：** Windows 系统托盘运行、开机自启动管理。
- ✓ **通知机制：** 通过 Server酱 发送流量超限与速率预警。
- ✓ **偏好管理：** 支持可视化修改配置参数。

### 进行中 (Active)
- [ ] **架构重构：** 将 `proxy_monitor.pyw` 拆分为独立模块，提高可测性。
- [ ] **测试驱动开发 (TDD)：** 引入 `pytest` 框架，编写针对网络、配置及状态管理的单元测试。
- [ ] **反馈增强：** 为关键失败点（如订阅解析失败）增加更友好的用户弹窗提示。
- [ ] **环境变量审计：** 增强自检逻辑，在缺少 Python 库或外部连接失败时提供明确的修复指引。

### 超出范围 (Out of Scope)
- **多平台支持：** 本阶段仅聚焦于 Windows 平台的健壮性优化。
- **UI 风格大改：** 维持现有暗色调风格，重心在于功能稳定性而非视觉重对。
