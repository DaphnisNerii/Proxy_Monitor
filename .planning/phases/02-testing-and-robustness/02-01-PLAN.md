# Plan 02-01: UI 线程修复与基础测试环境构建

## 1. 目标
解决 Phase 1 遗留的 UI 线程卡死 Blocking 问题，并搭建项目的自动化测试框架。

## 2. 实施步骤

**2.1 UI 线程通信桥接 (修复 UAT Test 3 Issue)**
- **文件**: `main.pyw` 和 `ui_components.py`
- **改动**: 
  - 核心原因：pystray 的菜单回调运行在后台线程，而在该线程中直接 `tk.Tk().mainloop()` 会导致事件队列僵死。
  - 修复：在 `main.pyw` 主线程预先创建一个隐藏的 `tk.Tk()` (如 `root.withdraw()`)，并进入 `root.mainloop()`。对于 pystray 和 monitor 的更新，如果需要更新 UI 或弹出设置窗口，使用 `root.after(0, callback)` 将任务派发回主线程执行。
  - `pystray.Icon.run()` 将改为运行在一个独立的守护线程中，从而不阻塞主线程的 `Tkinter` loop。

**2.2 测试框架配置**
- **文件**: `requirements-test.txt` (新建) 和 `pytest.ini` (新建)
- **改动**: 引入 `pytest`, `pytest-mock`。在 `pytest.ini` 配置默认的环境参数。

**2.3 第一批单元测试 (ConfigManager)**
- **文件**: `tests/test_config_manager.py`
- **改动**: 
  - 编写对 `ConfigManager.load_config()` 的测试。
  - 使用 `pytest-mock` mock 掉文件系统与 `winreg`，测试自启动选项同步是否能够正确调用 Registry API 且不会崩溃。

## 3. 验证原则
- `pytest tests/test_config_manager.py` 完全通过。
- 点击托盘 "修改首选项..." 能够正常弹出设置，输入框可交互，各项按钮可以被正常点击并保存配置消失。
