# Plan 03-02: 全局异常拦截与启动守护

## 1. 目标 (UX-02, UX-03)
确保程序在遭遇致命异常时不会"悄无声息地消失"，而是通过用户友好的弹窗告知问题所在，并在启动自检失败时给出明确的修复指引。

## 2. 实施步骤

**2.1 全局异常拦截弹窗**
- **文件**: `main.pyw`
- **改动**: 
  - 在 `main()` 的顶层 `try/except` 中，捕获未处理的异常后，使用 `tkinter.messagebox.showerror()` 弹出错误对话框，展示异常摘要和日志文件路径。
  - 同时将完整的 traceback 写入 `monitor.log`，便于事后排查。

**2.2 Pre-flight 失败的友好提示**
- **文件**: `system_utils.py` 和 `main.pyw`
- **改动**: 
  - 当 `PreFlightCheck.auto_repair()` 返回 `False` 时，不再简单地 `sys.exit(1)`。
  - 改为弹出一个信息丰富的错误窗口，列出缺失的依赖项，并提供手动修复命令（如 `pip install -r requirements.txt`）以及项目 GitHub Issues 链接。

## 3. 验证原则
- 手动制造一个 import 异常（如临时重命名 `config_manager.py`），运行 `python main.pyw`，应弹出友好的错误弹窗而非直接消失。
- 弹窗关闭后程序干净退出，不残留后台进程。
