# Plan 03-01: UX 增强与实时感知

## 1. 目标 (UX-01)
解决配置修改并保存后，用户缺乏直观“成功感”或“不知道接下来发生了什么”的现状。

## 2. 实施步骤

**2.1 保存后的状态栏汽泡通知**
- **文件**: `tray_app.py`
- **改动**: 
  - 为 `ProxyTrayIcon` 增加一个发送系统通知气泡（Bubble Notification）的方法，依赖于 `pystray.Icon.notify(message, title)`
  - 在 `main.pyw` 接收到 `on_settings_save` 回调后，触发通知：“配置已保存 - 正在刷新底层数据...”。

**2.2 立刻更新视图反馈**
- **文件**: `monitor_service.py` 和 `main.pyw`
- **改动**: 
  - 当 `refresh_event.set()` 被触发（提前打断旧轮询）进入新一轮请求时，一旦取得了最新数据，立刻借由 `icon_callback` 传递给托盘。这已在一部分存在，但需要更敏捷的响应。我们将在此处梳理确保网络刷新完毕后的首帧渲染顺畅无感。

## 3. 验证原则
- 保存“设置”后，桌面右下角会弹出 Windows 系统级的消息提示框：“配置已就绪，立刻更新了请求数据”。
