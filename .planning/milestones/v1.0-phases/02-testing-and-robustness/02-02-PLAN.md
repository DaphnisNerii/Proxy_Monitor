# Plan 02-02: 流量解析稳健性增强与核心 Mock 测试

## 1. 目标
通过完善单元测试覆盖 `monitor_service.py` 的异常网络用例，并在其中增加容错和退避逻辑，确保程序在无网或弱网环境下不会发生崩溃或频繁卡死。

## 2. 实施步骤

**2.1 Monitor 网络容错增强**
- **文件**: `monitor_service.py`
- **改动**: 
  - 当前虽然有简单的 `timeout`，但当连续多次抛出异常时，应当通过更完善的日志收集而不是直接忽略。
  - 对于网络超时的重试加入指数退避机制（Exponential Backoff）而非简单的 `time.sleep(5)`，提升网络恢复期的适应性。

**2.2 编写 TrafficMonitor Mock 测试**
- **文件**: `tests/test_monitor_service.py`
- **改动**: 
  - 使用 `pytest-mock` mock `urllib.request.urlopen`。
  - 测试正常解析出上传、下载和总流量的 Happy Path。
  - 测试 404/500 等异常请求，断言其正确返回 `None`。
  - 测试 Headers 缺失 `subscription-userinfo` 字段的情况。

**2.3 编写 SystemUtils 测试**
- **文件**: `tests/test_system_utils.py`
- **改动**: 
  - 使用 `mock` 隔离底层 Windows API 依赖。
  - 验证 `NamedMutex.acquire()` 在发生 `ERROR_ALREADY_EXISTS (183)` 时能正确返回 `False`。
  - 验证 `PreFlightCheck` 的修复逻辑执行路径。

## 3. 验证原则
- `pytest tests/` 全部通过。
- 在断网情况下运行程序并点击“立即刷新数据”，程序应表现为挂起而不崩溃，随后在恢复网络时继续。
